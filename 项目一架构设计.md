
项目名称：game-dev-blog（游戏开发技术博客）
 
一、核心技术栈
 
Vue3 + Vite + Pinia + Axios + TypeScript + Element Plus（隐含UI组件库，适配前端开发常规配置）
 
二、架构分层与核心目录
 
- 入口层：对应 src/main.ts，负责初始化Vue实例，挂载路由、Pinia，加载全局配置；
- 页面层：对应 src/views/，存放完整页面组件（首页、文章详情页、社区页、招聘页等）；
- 组件层：对应 src/components/，拆分通用组件+业务子组件（如讨论区、招聘区组件），实现代码复用；
- 路由层：对应 src/router/，配置页面路由规则，管理页面跳转（如首页→文章详情页的路由映射）；
- 状态管理层：对应 src/stores/，用Pinia存储全局数据（如用户登录状态、热门文章列表缓存）；
- 工具层：对应 src/utils/，封装Axios请求（request.ts）、数据处理函数，统一处理接口调用与错误；
- 静态资源层：对应 src/public/，存放图片、logo等静态资源，保障页面资源加载；
- 类型定义层：对应 src/types/，定义TypeScript接口类型（如文章数据类型），规范数据格式；
- 模拟数据层：对应 src/mock/，本地开发时提供模拟接口数据，无需依赖后端即可调试页面。
 
三、核心业务流程（以“用户查看文章详情”为例）
 
- 1. 用户通过路由点击文章标题，触发 router/index.ts 中配置的路由规则，跳转至 views/ArticleDetail.vue 页面；
- 2. 页面组件调用 utils/request.ts 封装的Axios请求，向后端获取目标文章数据；
- 3. 请求成功后，通过组件 props 或 Pinia 传递数据，渲染页面内的文章内容组件、评论区组件；
- 4. 若数据加载失败，工具层统一捕获错误并提示用户，保障交互体验。
 
四、架构设计亮点
 
- 1. 分层清晰：按“页面-组件-工具-路由”拆分，降低模块耦合，便于维护与扩展；
- 2. 复用性强：核心组件抽离至 components 目录，支持多页面调用（如社区模块组件可复用在首页与社区页）；
- 3. 规范统一：用TypeScript定义数据类型，Axios请求统一封装，减少重复代码与潜在错误；
- 4. 开发高效：Vite提升构建速度，Mock数据支持本地调试，降低前后端联调依赖。
